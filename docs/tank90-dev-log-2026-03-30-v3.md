# 90坦克大战 Web 开发记录（v3）

日期：2026-03-30  
版本：v3

## 版本目标（相对 v2）

- 关卡地形向原版 **Battle City Stage 1–10** 靠拢：采用公开 Battle City JS 克隆的 26×26 网格直译（砖/钢/水/草编码映射到 `B/S/W/G/.`），并保留基地护墙与 `withBaseRows` 行为。
- 体验与工程结构：
  - 方向键与空格在页面级 `preventDefault`，避免游玩时页面滚动。
  - 玩家出生在基地护墙上方安全位；地形生成时剔除基地 hitbox 内砖块，避免子弹无法命中基地逻辑体。
  - 首波生成延迟、敌人首射时间窗、较长出生护盾，以及 Stage 1 早期的活跃敌人数/开火概率抑制，缓解「开局即死」。
  - 外围 UI（标题、HUD 卡片、触控条等）接入全站语义色，画布区域保持 1:1 正方形（`data-testid="tank90-canvas"`）。
  - 产品文案统一 **STAGE**（`STAGE n`、`NEXT STAGE`），与 v2 的 `LV` 概念脱钩；实现层仍保留 `level` / `LEVELS` / `getLevelConfig` 命名，便于渐进重命名。
  - 将原先近千行单文件拆分为 `src/tank90/core/*`（类型、常量、状态、更新、渲染），`TankBattle90View.tsx` 仅保留页面编排与输入绑定。

## 计划与执行方式

- 基于计划文档推进并持续回写：
  - `/.cursor/plans/tank90-v3-iteration-plan_2a20f5dd.plan.md`
- 多轮 Generator / Evaluator 迭代（含严格评分与地图相似度讨论）；最终以 **无阻塞问题 + 自动化通过** 作为本轮收尾标准。

## 核心改动（已落地）

### 1) 关卡数据（Stage 1–10）

- `src/tank90/levels.ts`：`STAGE_01` … `STAGE_10` 地形常量 + `LevelConfig` 列表；每关仍配置 `enemyQueue` / `spawnDelaySec` / `enemiesTotal`。
- 注释标明网格来源与单元编码规则，便于后续与 StrategyWiki 原图做叠图校对。

### 2) 游戏逻辑与状态（core）

- `core/state.ts`：`createState`、`spawnPlayer`、`terrainToBlocks`（基地格内不铺地形块）。
- `core/update.ts`：帧更新、生成、AI、碰撞；含 `elapsedMs`、Stage 1 早期安全窗口与生成/射击节奏控制。
- `core/render.ts`：Canvas 绘制；开屏横幅使用 `STAGE` 文案。
- `core/types.ts` / `core/constants.ts`：世界尺寸、出生点、早期平衡常量等。

### 3) 页面与输入

- `TankBattle90View.tsx`：`keydown`/`keyup` 对方向键与 `Space` 非被动监听并 `preventDefault`（排除输入类元素）；触控与 debug 面板行为与 v2 一致。
- `TouchControls.tsx`：语义色类名与主题一致。

### 4) 自动化测试

- `e2e/tank90.spec.ts`（5 条）：
  - 主流程（debug）：`STAGE` HUD、`NEXT STAGE`、暂停、强输强赢、滚动断言。
  - 正方形画布 + 主题外围 class 变化。
  - 移动端控件与菜单避让。
  - 多轮 pause/restart/win/lose 与多关推进。
  - 持续输入 soak 与无运行时错误。

## 关键问题修复记录

- 页面滚动：方向键/空格默认行为被拦截，避免 `/tank90` 游玩时整页滚动。
- 开局暴毙：首敌生成与首射延后、出生护盾延长、Stage 1 早期减载与降开火率。
- 基地与地形：`terrainToBlocks` 跳过基地矩形内格子，与 `base` 碰撞一致。
- 单文件过大：逻辑迁入 `core/`，主视图瘦身。
