# 90坦克大战Web 开发记录（概要）

日期：2026-03-26  
版本：v1

## 游戏复刻定义（MVP）

- 游戏类型：复古像素风的“坦克大战/ Battle City ”复刻（网页端、Canvas）。
- 目标体验：
  - 操控玩家坦克，四向移动 + 发射
  - 保护基地（鹰/要塞），基地被击中则失败
  - 每关需要清空敌军坦克数量后进入下一关
  - 砖墙/障碍可被子弹破坏（体现“打穿墙体”的核心手感）

## 操作与状态

- 键盘优先（MVP）：
  - `WASD`：移动
  - `Space`：射击（按住会以重装时间节流连发）
  - `P`：暂停/恢复
- 游戏状态（HUD）：
  - `ready`：等待开始
  - `running`：运行中
  - `paused`：暂停中（遮罩 + 文案）
  - `won`：通关态（显示 `NEXT LV`）
  - `lost`：失败态（显示 `GAME OVER / LOST | PLAYER DEAD`）

## Web 技术栈选择（为何这样做）

- 渲染层：`<canvas>` + `requestAnimationFrame`
  - 优点：帧率可控、像素/复古风表达直接、对工程依赖少
- 工程层：沿用仓库现有的 `React 19 + Vite + Tailwind + react-router-dom`
- 运行时状态管理：
  - 游戏主状态放在 `useRef`，避免每帧 `setState` 触发 React 重渲染
  - 需要时只同步少量 HUD 信息到 React state

## 验收记录（Evaluator）

### 首轮验收：不通过

- 通过项
  - 入口/路由正确、`P` 暂停、基地保护与 `GAME OVER`、敌方生成稳定、视觉方向正确
- 不通过项
  - 玩家可见性/可验收性不足：`WASD/Space` 难稳定确认
  - `NEXT LV` 的语义不严谨：非通关状态下仍可看到/操作
  - `RESTART` 后 HUD 有旧状态瞬态残留

### 第二轮验收：通过

- 通过项
  - `START/RESTART` 后玩家可见可控，HUD 显示 `PLAYER ALIVE`，战斗反馈可观察
  - `P` 暂停/恢复生效（`PAUSED` 遮罩/文案，恢复 `RUNNING`）
  - 基地被击中进入失败态：HUD 同步为 `LOST | PLAYER DEAD`
  - `RESTART` 后状态一致性良好，无旧 `LOST/WON` 残留
  - `NEXT LV` 约束正确：仅通关态出现且可用
  - 视觉风格与街机一致，敌方生成与战斗节奏可用

## Improvements 落地（维护性增强）

1. 开发调试面板（dev-only）
   - 触发：仅开发环境且带 `?debug=1`（例如 `/tank90?debug=1`）
   - 作用：快速闭环主链路与回归
   - 按钮：`FORCE_WIN` / `FORCE_LOSE` / `CLEAR_ENEMIES` / `TOGGLE_PAUSE`

2. 轻量事件日志 overlay（dev-only）
   - 记录并展示：状态迁移（如状态机变化）、关卡变化、击杀计数变化、玩家/基地生死变化
   - 使用 ring-buffer 保留最近事件，方便定位“状态是否按预期走”

3. Playwright E2E（主链路）
   - 只跑 Chromium（避免在当前 macOS/架构上安装 WebKit 的限制问题）
   - 覆盖主链路：`START -> PAUSE/RESUME -> FORCE_LOSE -> RESTART -> FORCE_WIN -> NEXT LV`
